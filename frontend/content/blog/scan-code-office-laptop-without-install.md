Corporate IT often blocks:

- Chrome extensions
- Desktop apps
- Unknown API endpoints

**CamToCode** runs in a normal browser at **[camtocode.com](https://camtocode.com)** — like visiting any HTTPS website.

## Workflow for locked laptops

### Option A — Phone scans laptop screen

1. Display code on your laptop (VS Code, browser, PDF).
2. Open [camtocode.com/app](https://camtocode.com/app) on your **phone**.
3. Point camera at screen → capture → get clean code.

### Option B — Scroll Automation (best for long files)

1. Open **[camtocode.com/scroll](https://camtocode.com/scroll)** on the laptop.
2. Load the same file via file picker (stays local — no upload).
3. Auto-scroll sections; phone captures each part with Auto Re-capture.

Scroll Automation uses **zero backend calls** for display and scroll.

## What still needs internet

- **Phone OCR** uses CamToCode AI on Railway (same as any cloud OCR).
- **Scroll page** works offline after first load (don't refresh).
