HOW IT WORKS:
================================================================================

OLD (What you had before):
  Line-level voting: pick the entire line that appears most
  Problem: if 3/5 frames have same error, you're stuck with it

NEW (What you have now):
  Character-level voting: for each position, pick the character that appears most
  Benefit: combines best characters from all frames, fixes errors automatically

EXAMPLE:
  Input (5 frames):
    Frame 1: "def foo():"
    Frame 2: "def f00():"  ← Error: 'o' → '0'
    Frame 3: "def foo():"
    Frame 4: "def foo():"
    Frame 5: "def fo0():"  ← Error: 'o' → '0'

  Character voting at position 5:
    Votes: [o, 0, o, o, 0]
    Winner: 'o' (appears 3/5 times) ✅

  Output: "def foo():" ✅ CORRECT!

================================================================================
ALGORITHM DETAILS:
================================================================================

Input: texts = ["frame1_ocr", "frame2_ocr", "frame3_ocr", "frame4_ocr", "frame5_ocr"]

Process:
  1. Split each text into lines
  2. For each line index (0, 1, 2, ...):
     a. Collect all versions of that line from all frames
     b. For each character position (0, 1, 2, ...):
        - Build Counter of characters at this position
        - Pick character with highest count (votes)
        - Calculate confidence (e.g., 3/5 = 60%)
     c. Build consensus line from winning characters
  3. Return joined lines

Time Complexity: O(n * m) where n = frames, m = average line length
Space Complexity: O(n * m) for storing frame data
